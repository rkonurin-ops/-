# Анализ пользовательского поведения Яндекс Афиши

## Описание проекта
Исследовательский анализ данных билетного сервиса. 
Цель — выявить факторы влияющие на возврат пользователей и совершение повторных заказов.

## Данные
- База данных PostgreSQL, схема `afisha`
- Период: июнь–октябрь 2024 года
- 5 таблиц: `purchases`, `events`, `city`, `regions`, `venues`

## Стек технологий
- Python (pandas, matplotlib, seaborn, phik, sqlalchemy)
- PostgreSQL
- Jupyter Notebook

## Структура проекта
- `Яндекс_Афиша_проект.ipynb` — основной ноутбук с анализом
- `requirements.txt` — зависимости проекта
- `.env` — параметры подключения к БД (не включён в репозиторий)

## Основные выводы
- 61.7% пользователей совершают повторный заказ
- Наибольшая лояльность у пользователей покупающих 2–3 билета (74% возврат)
- Короткий интервал между заказами (до 10 дней) — ключевой признак лояльного пользователя
- Тип мероприятия, устройство и билетный оператор слабо влияют на возврат

## Запуск проекта
1. Установить зависимости: `pip install -r requirements.txt`
2. Создать `.env` файл с параметрами подключения к БД
3. Запустить `Яндекс_Афиша_проект.ipynb`
